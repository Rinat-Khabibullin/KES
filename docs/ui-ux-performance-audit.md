# UI/UX и Performance Аудит

Дата аудита: 2026-07-19.

## Контекст

Сайт работает как React 19 + Vite SPA с React Router, лендингом `/`, калькулятором `/calculator`, Vercel Functions и GigaChat-чатом. Аудит выполнен по локальной production-сборке `npm run build` и `vite preview`, потому что это воспроизводимый production-like режим. CDN/реальные Vercel headers и RUM-метрики пользователей в этой среде недоступны.

## UI/UX Проблемы

- Визуальная система смешивает старые токены (`--surface`, `--electric-blue`, `--line`) и прямые rgba/hex-значения. Семантические цвета не отделены от декоративных.
- Кнопки имеют разные классы и размеры без единой шкалы: `.button`, `.button--small`, `.mobile-cta__button`, icon buttons в чате и калькуляторе.
- На Hero одновременно три CTA, все визуально конкурируют. Основной сценарий должен быть один, вторичный - калькулятор/фото.
- Карточки с hover-эффектами применяются и к некликабельным элементам, что создаёт ложную кликабельность.
- Массовый reveal-effect ищет много DOM-элементов после первой отрисовки. Контент не скрывается полностью, но это лишняя работа на старте.
- Hash navigation использует три таймера и смещается к вложенному заголовку, из-за чего переход может приводить не к началу секции.
- Быстрые вопросы чата постоянно занимают место под историей, не сворачиваются после начала диалога и всегда отправляются через API.
- При сетевой ошибке нет явной кнопки повторить последнее сообщение.
- Skip-link отсутствует.

## Мобильный UX

Проверенные риски:

- Header на 320 px уже был исправлен ранее, но система размеров всё ещё хрупкая из-за разных классов кнопок.
- Нижняя панель `Позвонить | Фото | Чат` полезна, но визуально живёт отдельно от общей button-системы.
- Чат на мобильном работает как full-screen, но быстрые вопросы могут создавать лишнюю высоту и вытеснять историю.
- Калькулятор удобнее после свёрнутого мастера и порционной отрисовки, но он всё ещё попадает в initial bundle лендинга.

## Accessibility

Положительно:

- Основные landmarks и heading hierarchy в целом корректны.
- Mobile menu имеет `aria-expanded` и `aria-controls`.
- Chat dialog закрывается по Escape, фокус частично возвращается.

Что улучшить:

- Добавить skip-link.
- Hash navigation должна переводить фокус на секцию/заголовок после перехода.
- Для modal/chat желательно усилить focus management.
- Декоративные иконки в кнопках должны иметь текст рядом или `aria-hidden`.
- Быстрые подсказки чата должны иметь понятное управление закрытием/возвратом.

## Performance Проблемы

- Initial JS chunk содержит калькулятор, каталог, поиск, расчёт, чат и все страницы.
- Нет route-based code splitting.
- Modal импортируется сразу, хотя нужен только при клике по фото.
- ChatWidget импортируется сразу, хотя пользователь может вообще не открывать чат.
- CSS большой и глобальный; есть дубли токенов, старые `.price-grid/.price-card` стили и много прямых декоративных значений.
- Бесконечные декоративные анимации (`pulse-line`, `ribbon-slide`) не критичны, но не нужны на мобильных и могут быть дешевле.
- Public prices API можно кэшировать дольше, потому что каталог версионирован. Chat API уже `no-store`.

## Решение

Минимально сложный надёжный путь:

- Оставить Vite SPA, не мигрировать на Next.js.
- Добавить route-based code splitting для `/calculator`.
- Лениво грузить modal и полноценный chat widget.
- Ввести семантические design tokens и согласовать button classes.
- Заменить timer-based hash scroll на устойчивый `ScrollToHash`.
- Быстрые вопросы обрабатывать локально через данные сайта и каталог.
- Уменьшить стартовую JS-работу и сохранить Hero видимым сразу без полноэкранного skeleton.

## Реализовано

- Введены семантические design tokens: `--color-bg`, `--color-surface`, `--color-surface-elevated`, `--color-text`, `--color-text-muted`, `--color-border`, `--color-primary`, `--color-primary-hover`, `--color-accent`, `--color-success`, `--color-warning`, `--color-error`.
- Добавлены системные токены отступов, типографики и размеров кнопок: `sm` около 42 px, `md` 48 px, `lg` 56 px, `icon` 48 px.
- Hero оставлен с одним главным CTA и одним вторичным CTA: бесплатная консультация и расчёт сметы.
- Добавлены skip-link и фокус на целевую секцию после hash-перехода.
- Hash navigation вынесена в `ScrollToHash`; удалён подход с конкурирующими таймерами.
- Reveal-инициализация отложена до idle, основной контент не скрывается до JS.
- `/calculator`, `ChatWidget`, `Modal` и `NotFoundPage` загружаются лениво.
- Полный каталог цен убран из initial bundle лендинга; для лендинга используется лёгкий typed snapshot популярных цен, который тестами сверяется с каноническим каталогом.
- Быстрые вопросы чата стали типизированными actions и отвечают локально, если ответ можно получить из данных сайта/каталога.
- Быстрые подсказки скрываются после выбора или ручного сообщения и возвращаются кнопкой `Подсказки`.
- Добавлена кнопка повтора последнего сообщения при сетевой ошибке.
- Исправлена гонка lazy-загрузки чата: первый клик по launcher теперь открывает панель после монтирования `ChatWidget`.
- Добавлены `robots.txt` и `sitemap.xml`; SEO Lighthouse поднялся с 92 до 100.
- Добавлены immutable cache headers для `/assets/*` и `/works/*`; `/api/chat` и `/api/health/chat` оставлены `no-store`.
- `/api/prices` получил публичный cache с `ETag` по версии каталога; `/api/prices/search` получил короткий публичный cache.

## Playwright Проверки

Production preview проверен на `360x800`, `390x844`, `768x1024`, `1440x900`.

- `/` и `/calculator`: горизонтального overflow нет, Hero видим.
- `/calculator` -> `/#services`: секция становится под sticky header, фокус на `#services`.
- Мобильное меню открывается и доступно как `Мобильная навигация`.
- Быстрый вопрос `Сколько стоит вызов электрика?`: локальный ответ, 0 запросов к `/api/chat`, подсказки сворачиваются.
- Свободный вопрос при 500 от `/api/chat`: запрос выполняется один раз, появляется кнопка `Повторить последний вопрос`.
- Скриншоты: `/tmp/kes-screens`.

## Ограничения

- `npm run build:vercel` локально не завершился из-за недействительного Vercel token: `The specified token is not valid`. Скрипт исправлен на `vercel build --yes`; после `vercel login` или нормального `VERCEL_TOKEN` команда должна пройти.
- Фото портфолио сейчас достаточно лёгкие, поэтому отдельные thumbnail/full варианты не добавлялись. При загрузке новых тяжёлых фото нужен генератор responsive images.
- CSS всё ещё остаётся одним большим файлом. Он структурирован токенами и секциями, но физическое дробление стоит делать следующим шагом, чтобы не смешивать рефакторинг стилей с текущей оптимизацией.
